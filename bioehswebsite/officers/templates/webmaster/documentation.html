Site Basics

Setup

  • This website was built using Django, a Python-based web framework. Django basically provides a way to make a website that includes multiple "apps", or you might have heard them called "web-apps," and it uses a paradigm called "Model-View-Controller." This is a common design paradigm that app developers and web developers like to use a lot to separate the user interface from the logic and functionality of a project.

  • The wiki we have is through MediaWiki, PHP-based wiki, and it is also hosted under the same domain.

  • The frontend is all HTML, CSS, and Javascript, and the backend is in SQLite3. BUT Django provides a great admin site and a nifty Python API to the database so you should never (need to) edit the SQL database directly.

  • This site along with the wiki is hosted by the OCF.

Resources for Getting Started

Before continuing, you should take time to learn Python, HTML, and Django (at least a little each). You may not need to use any of them because the admin portal should give you full functionality for most things but it will make your life easier if you know these things to some extent.

Here are the most helpful resources for learning Python/Django/HTML development:

  • Django Tutorial:

    Go through this tutorial for building a simple Django polls app. You should really do this, but at the very least you should read it. It will help you understand the organization of this site.

  • Codecademy HTML Course:

    Literally takes 1 hour and is enough to get you up and running with the frontend of this site.

  • Read through all the .py files in this site, understand especially how the urls.py, views.py, and models.py files interact in each app directory. Look through some of the .html files.

  • The HTML templates in this site use something very powerful called Jinja templating, which is a way to embed Python variables in HTML. Check out officers/templates/officers.html or any of the files in officers/templates/accounts to see what I mean. Skim the Jinja reference here.

Organization of Site

The site is divided into many different 'apps,' as described in the Django documentation on reusable apps.

  • contact: an app that handles the functionality in the Contact Us page.

  • events: an app that handles the calendar.

  • home: an app that handles basically all the non-complicated, static rendering of the site, for pages that mostly just display information and have little other functionality.

  • industrial: an app that handles the industrial relations. and things like adding sponsors and storing that info.

  • officers: an app that handles all the user and account-related stuff. 'Officers' is a misnomer as this app handles user authentication for officers, candidates, general members, alumni, and anyone with an account.

Each app contains its own models, views, templates, static files like CSS files and Javascript files, and migrations â€” which are executables that tell Django how to update the database whenever we add models or change them.

The bioehs directory is not an app itself but it is the heart of the site and houses all the other applications. It includes things like the settings and configurations for the whole project and tells Django in which apps to look for different url's.

The assets folder is where any user-uploaded (including admin-uploaded) media will go. This is essentially for non-static content like profile pictures or flyers that could change.


Site Administration

To keep the site maintained, you will need to update information each semester. This includes adding users, updating user profiles (such as for instance, updating candidate requirements), changing site content (such as updating the BioEHSC pages), and updating images, links, flyers, etc. Luckily the admin site is set up to handle such things.
